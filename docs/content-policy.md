# Content Policy

AI Tools History Archive should be neutral, evidence-based, and careful with uncertainty.

## Core Rules

- Prefer public source-backed statements.
- Separate confirmed facts from uncertain observations.
- Use confidence labels when the available evidence is incomplete.
- Avoid sensational wording.
- Avoid assigning motives unless a public source explicitly supports the statement.
- Treat the archive as a registry, not a review site.

## Allowed Content

Allowed record content includes:

- Public product names
- Public operator names
- Public launch dates
- Public lifecycle event dates
- Public source links
- Archived source links
- Status labels
- Current-state notes
- Category labels
- Evidence and reliability labels
- Known limitations

## Avoid

Avoid wording that turns incomplete evidence into a final judgment.

Do not use unsupported claims about misconduct, intent, quality, or blame.

Prefer wording such as:

```text
confirmed shutdown
announced shutdown
appears inactive
website unavailable
domain redirected
no official notice found
status unknown
needs review
```

## Uncertainty

When evidence is incomplete, use:

```text
unknown
unverified
needs_review
low confidence
```

Do not hide uncertainty. It is better for a record to be incomplete and clearly labeled than to overstate what is known.

## Corrections

Public corrections should be welcomed when they include useful sources. A correction can update status, dates, summary, category, URL status, evidence, or related records.
