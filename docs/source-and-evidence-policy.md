# Source and Evidence Policy

AI Tools History Archive is evidence-first. Records should be backed by public sources and should make clear what each source supports.

## Preferred Source Types

Highest priority:

- Official announcements
- Official documentation
- Official support pages
- Official changelogs
- Press releases
- Regulatory or court documents when relevant

Useful secondary sources:

- Credible media reports
- Archived pages
- Company social posts
- GitHub releases or repository notes
- Public forum or user reports when clearly labeled as lower-confidence material

## Evidence Fields

Each evidence item should include:

- Title
- URL
- Source type
- Publisher or operator when available
- Published date when available
- Accessed date
- Archived URL when useful
- What claim the source supports
- Reliability label

## Reliability Labels

```text
high
medium
low
unknown
```

General guidance:

- `high`: official sources, primary documents, or highly reliable public records
- `medium`: credible secondary reports, archived pages, or multiple mutually supporting sources
- `low`: user reports, forum posts, or sources that need more review
- `unknown`: not yet evaluated

## Event-Level Evidence

Evidence should be linked to specific events when possible.

Example:

```text
Record: Example AI
Event: Rebrand announced
Evidence: Official blog announcement
Supports: rebrand date, new product name, operator statement
```

## Archive Links

Archived URLs are recommended when:

- The original page may change
- The product is no longer available
- The original domain redirects
- The source is central to a major lifecycle claim

## Public Link Safety

If an original domain appears inactive, repurposed, or unsafe, the public UI should prefer archive links and clear text labels rather than encouraging users to visit the old domain.

## Corrections

A correction should ideally include:

- Record name
- What should change
- Evidence URL
- Date of the event if known
- Reason for the correction
