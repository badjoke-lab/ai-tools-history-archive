# UI Guidelines

AI Tools History Archive uses a clean white-background archive style.

## Direction

The interface should feel like:

```text
public archive
documentation site
neutral registry
source-linked database
```

## Required Baseline

- White background
- Dark text
- Thin borders
- Clear tables and cards
- Calm spacing
- Simple badges
- Clear evidence panels
- Clear timeline blocks
- Responsive mobile layout

## Avoid

Do not make the UI look like:

- A trading terminal
- A dark terminal
- A graveyard joke site
- A hype-heavy SaaS landing page
- A ranking site
- A crypto dashboard
- A neon AI tool directory

## Core Components

Planned components:

- Header navigation
- Search field
- Filter panel
- Record card
- Record table row
- Status badge
- Confidence badge
- Timeline item
- Evidence card
- Current-state block
- Known-unknowns block
- Related-records block
- Stats cards

## Tone

The UI should communicate that the archive is careful, factual, and source-linked. It should not imply that a product is bad, failed, or blameworthy unless the record is directly supported by public evidence and neutral wording.
